## 一、项目方向确认

结合前期课程理解、个人选题筛选与最新一轮项目收敛，小组最终确定本学期主攻方向为：

**面向中国大陆求职市场的 AI 驱动岗位匹配与职业助手平台（Jobro China）**

确定这一方向，主要基于以下五点考虑：

1. **用户痛点真实且高频：** 中国大陆学生、应届生和职场早期用户在求职过程中普遍面临岗位信息分散、秋招春招窗口短、短期机会易错过、平台匹配质量不稳定等问题。
2. **课程落地性较强：** 项目可以围绕 `简历解析 + 偏好建模 + 岗位聚合 + AI 匹配 + 聊天反馈` 搭建可演示的 MVP，适合在课程周期内形成完整方案。
3. **技术壁垒清晰：** 项目涉及非结构化简历与岗位描述理解、跨来源岗位标准化、推荐排序、反馈学习和推荐解释性，能体现“硬科技 + 垂直场景”的技术深度。
4. **市场空间足够大：** 中国大陆求职市场规模大、信息渠道复杂、校招与实习节奏明确，既有真实需求，也便于后续延展商业模式与增长策略。
5. **成果便于量化：** 用户激活率、匹配接受率、留存率、首批推荐时延和转化率等指标都可清晰衡量，适合课程答辩与可行性论证。

## 三、核心问题与应用场景

### 1. 核心问题

当前中国大陆求职场景主要存在以下问题：

1. **信息高度碎片化：** 用户需要在高校就业网、企业校招官网、BOSS 直聘、智联招聘、前程无忧、牛客网、微信群、公众号、小红书等多个渠道反复搜索，成本很高。
2. **匹配质量不稳定：** 传统平台往往更多依赖静态标签和关键词匹配，难以持续理解用户真实意图、能力边界和偏好变化。
3. **机会窗口很短：** 校招、实习和内推岗位常常存在发布时间集中、截止时间短、更新节奏快的问题，用户容易错过高质量机会。
4. **反馈无法形成闭环：** 用户在“感兴趣 / 不感兴趣 / 想看更多某类岗位”等行为上的信号，通常不能被现有系统充分吸收和迭代。
5. **早期求职者缺少陪伴式助手：** 很多学生和应届生并不只是缺岗位，更缺一个能持续理解自己、帮助收敛方向并减少焦虑的职业助手。

### 2. 主要应用场景

本项目适用于以下高频求职场景：

* 本科生和硕士生的秋招、春招岗位发现与筛选
* 在校生的实习机会搜索、比较与持续跟进
* 毕业后 0 到 3 年用户的首份正式工作或初次转岗探索
* 信息密度较高、岗位更新较快的知识型岗位场景
* 需要快速聚合高质量招聘信息并持续修正偏好的用户群体

## 五、技术实现思路

本项目初步采用“岗位聚合 + 用户理解 + 混合匹配 + 反馈闭环”的技术路线：

1. **数据获取层：** 从高校就业网、企业校招官网、公开岗位页面和人工整理表格中获取岗位信息，形成首版岗位数据库。
2. **用户理解层：** 通过简历解析、偏好表单和后续聊天输入，构建用户的结构化画像与动态偏好表示。
3. **匹配计算层：** 结合岗位标签、规则过滤、向量召回或相似度匹配，以及大模型辅助重排，生成推荐结果。
4. **交互反馈层：** 用户可查看岗位卡片、点击投递、标记不感兴趣或通过聊天修改偏好，系统持续更新用户画像。
5. **分析与增长层：** 通过事件埋点记录激活、留存、点击和转化等关键指标，为后续产品迭代提供依据。

从硬科技属性上看，项目的难点不在于简单抓取或关键词搜索，而在于：

* 非结构化简历和岗位描述的稳定理解
* 多来源岗位信息的统一标准化与去重
* 用户意图与岗位要求之间的高质量匹配
* 推荐结果的解释性与反馈闭环能力
* 个人信息保护与推荐逻辑之间的平衡

## 八、当前风险与应对思路

在项目初步规划阶段，已经可以预见以下几类风险：

1. **岗位数据与合规风险：** 不同招聘渠道对信息展示、采集和再分发的容忍度不同，简历和行为数据本身也涉及个人信息保护要求。
2. **推荐质量冷启动风险：** 在缺少大量历史行为数据的情况下，首版匹配效果可能不稳定，影响用户第一印象。
3. **市场范围过大风险：** 中国大陆求职市场非常大，如果首批用户或岗位范围不收敛，容易导致产品能力发散。
4. **触达方式不确定风险：** 用户更习惯哪类注册方式和消息通道，仍需调研核验，过早拍板可能造成产品形态错位。
5. **时间投入有限：** 三位成员每周投入时间均约 8 小时，因此必须优先聚焦课程期内真正能跑通的最小链路。

对应的基本应对策略是：

* 先聚焦校招/实习与职场早期场景，不做全量市场
* 先用“高质量人工导入 + 少量公开岗位来源”搭建首版数据库
* 先采用“规则过滤 + 标签匹配 + 模型辅助重排”的稳健路线
* 先验证一个外部消息通道，不同时铺开所有触达方式
* 先保证核心推荐链路可运行，再扩展增长与商业化功能
